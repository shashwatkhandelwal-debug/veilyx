Proofs,
not documents.

Veilyx lets your users prove who they are without handing over their Aadhaar. Verification happens entirely on-device. You get a cryptographic proof. Nothing is stored.

01

Quick Start

1
Register your company
Get your API key in one call. No account creation, no dashboard.
bash 
curl -X POST https://web-production-fe8772.up.railway.app/company/register \
  -H "Content-Type: application/json" \
  -d '{"company_name": "YourApp"}'
response 
{
  "company_id": "uuid",
  "company_name": "YourApp",
  "api_key": "your_api_key_here"  ← save this
}
2
Install the SDK
Drop the SDK into your Android or iOS project.
// build.gradle
implementation 'com.veilyx:sdk:1.0.0'
// Podfile
pod 'Veilyx', '~> 1.0'
npm install veilyx-react-native
3
Verify a user
Fetch a nonce, request a proof, verify it. That's the entire flow.
// 1. Fetch nonce from your backend
val nonce = fetchNonce()

// 2. Request proof on-device
val proof = veilyx.requestProof(
    companyName = "YourApp",
    checks = listOf("age_above_18"),
    nonce = nonce,
    aadhaarXml = xmlString
)

// 3. Send proof to your backend to verify
val result = api.verify(proof)
// result.valid == true → user is verified
// 1. Fetch nonce from your backend
let nonce = await fetchNonce()

// 2. Request proof on-device
let proof = try await veilyx.requestProof(
    companyName: "YourApp",
    checks: ["age_above_18"],
    nonce: nonce,
    aadhaarXml: xmlString
)

// 3. Send proof to your backend to verify
let result = try await api.verify(proof)
// result.valid == true → user is verified
// 1. Fetch nonce
const { nonce } = await fetch(`${BACKEND_URL}/nonce`, {
    headers: { 'X-API-Key': API_KEY }
}).then(r => r.json());

// 2. Request proof on-device
const proof = await Veilyx.requestProof({
    companyName: 'YourApp',
    checks: ['age_above_18'],
    nonce, aadhaarXml
});

// 3. Verify on your backend
const result = await fetch(`${BACKEND_URL}/verify`, {
    method: 'POST',
    headers: { 'X-API-Key': API_KEY },
    body: JSON.stringify(proof)
}).then(r => r.json());
// result.valid == true → user is verified
02

Authentication

All authenticated endpoints require your API key in the request header.

header
X-API-Key: your_api_key_here
⚡ Your API key is tied to your company name. The requested_by field in every proof must match your registered company name exactly.
